3、具体原理

算力＋存力：优化基因测序的关键所在 GenerationSequencing

一级分析：将需要进行测序的生物样本送入基因测序仪中，由测序仪进行WGS(全基因组测序)、WES(外显子测序)等测序操作，测序结果输出到可用于计算机进行分析的FASTQ文件中。FASTQ文件类似一个文本文件，每两行或三行记录一个DNA片段的碱基序列信息，测序质量信息等。 完整的DNA测序样本在进行测序前会被用超声波击碎为上千万条。

二级分析：这个过程现在通常由高性能处理集群完成。首先通过高性能服务器将测序仪生成的FASTQ文件(盗版书)与参考基因组文件(正版书)进行比对生成BAM文件。然后对BAM文件进行排序和去重，生成可以用于变异检测的BAM文件。

在最后，将BAM文件(有正确页号和行号的盗版书)与参考基因组文件(正版书)再次进行比对，找出BAM文件中所有与参考基因组文件不同之处，并将这些不同(变异)写入到VCF文件中。

三级分析：医学专家或科研人员通过分析服务器和专业数据库对VCF文件进行解读，给出疾病解读报告或者基因关联分析报告。疾病解读报告主要用于临床，比如某种基因突变是否可以使用某种靶向药物等。基因关联分析报告主要用于科学研究，比如某种基因突变会带来某种生理现象或者疾病等。

3、两序列比对概述

  • 残基：一个核苷酸残基 = 一个碱基 + 一个糖分子 + 一个磷酸基团

  • 残基的四种状态：匹配、替换、插入、删除（插删相当于在序列中引入了起占位作用的空位字符“-”）

  • 寻找最佳对齐方法：建立比对优劣的量化方法（打分模版——要求简单并具有生物学意义）

    • 最常用打分矩阵：PAM系列矩阵；BLOSUM系列矩阵

    • 常见罚分模型：固定空位罚分模型、仿射空间罚分模型

    • 打分准测

      • 将评估复杂序列比对转化为评估每一列的比对，总得分为各列得分总和

      • 为四种状态赋分（由于插入和删除通常赋负分，也称为“罚分模型”）

        e.g. S(a,b)=S(b,a)为匹配或替换的评分；S(a,-)=S(-,a)为插入或删除的得分。对于长度为L的连续空位字符形成的空位区，固定空位罚分模型的打分为f(L)=-d*L；而仿射~得分为f(L)=-g-d*(L-1)，其中g和d为非负常数。

2、Accelerating Genome Analysis: A Primer on an Ongoing Journey

基因组分析基本上是从一个已知的测序片段**比对定位（Read mapping）**过程开始的，在这个过程中，将生物基因组的测序片段与参考基因组进行比对（sequence alignment）。 瓶颈在于read mapping，如今基因测序比分析快得多。

read: small random fragments of an organism’s DNA sequence. (hundred to million bases)

baes: 碱基

文章分析read mapping的发展方向，展示高性能计算架构，即最先进的 1）算法 2）软硬件协同方式 3）硬件加速方法；展示结构是按照基因分析流程的三个步骤，针对每个步骤介绍加速方法。

基因组拼接（Genome assembly）是目前快速准确确定个体整个基因组的瓶颈（bottleneck），这是因为拼接需要复杂的算法和大量的数据集。

read mapping中read与reference genome比对的错误来源（测序错误、个体变异）；故使用近似字符串匹配算法ASM（approximate string matching），过去通常使用DP（dynamic programming，动态规划）算法作为软件在CPU上，占用了70%的处理时间[^1]，例如【minimap2[^2]】（一个read mapper）；并使用基因变异回溯（genomic variant calling）算法找到不同。

【最先进的CPU上read mapper[^3]综述】，问题：

  • cpu和主存间数据传输；
  • 测序和分析速度矛盾，相差几十倍；
  • 增加CPU数提高速度就会提高功耗，且比对未知基因时依次比对的工作量更大，云计算对数据保护不友好。

READ MAPPING

找相似子序列（E edits，其中E是距离阈值edit distance threshold，为将一个序列转换为另一个序列所需的最小更改次数。 编辑操作包括删除、插入和替换一个或两个序列中的字符）；前置操作indexing and filtering以减少比对片段数

具体过程：1）划分Read和Reference Genome的 substrings (seeds，通过哈希索引或精确匹配算法（如k-mer匹配）找到的完全相同的短序列片段) 并索引编号（即图中k-mers，为长度为k的滑动窗口），定位相同片段 2）过滤器（pre-alignment filters）过滤不相似（超过E）的配对，减少序列比对算法使用次数 3）解决 一个ASM（approximate string matching）问题，使用DP算法寻找使比对分数（alignment score，自定义，先定位edits，再基于the number of edits and/or matches）最大的前缀（prefixes，即从开头到某一位置的子串）

输出结果到 sequence alignment/map (SAM) file

注：DP算法提供最精确值但时空复杂度为O(n^2)，n为序列长

  • ACCELERATING INDEXING

    采用seed以避免对整个Genome使用ASM，只有read上的关联location才使用ASM。最大的挑战在于选择seed的长度和产生索引的数量。过短需要检查的location太多，太长匹配率下降。

    加速方法：减少seeds数， 通过从基因组区域内的一组相邻种子中寻找最小代表性种子集合(称为最小化器，minimizers) 。可以通过强制排序来计算代表集（按字典顺序或哈希值）？并使用先进数据结构来索引（例如【FM-index[^4]】，全文索引的压缩表示和不解压下的查询（query）；优点：可查询任意长度种子、比minimap2的存储占用小1倍。但压缩导致定位速度较经典算法慢，改进——【BWA-MEM2[^5]】）

    • Reducing Data Movement During Indexing

      【RADAR[^6]存算一体ASIC】

  • ACCELERATING PRE-ALIGNMENT FILTERING

    根据E edits过滤， Pre-alignment filter 的四种主要方法： 1) the pigeonhole principle; 2) base counting; 3) q-gram filtering; or 4) sparse DP.

    一般选用3)、4），其性能与read长度线性相关，且与edit distance无关

    • 信鸽原理Pigeonhole Principle

      长度为m的read， the read and reference segment are considered similar if they share at most E+1 nonoverlapping subsequences（differ by E edits）, with a total length of at least m-E.

      【Shouji‘s work[^1]】（其他工作：SHD on FPGA、GenCache for cache）

    • base counting filter

      根据各碱基总数过滤，差别大于E则过滤。

    • q-gram filtering

      一般long read mappers使用、

      q-gram: all of the sequence’s possible overlapping（重叠的） substrings of length q.

      例如，对于字符串 "hello"，它的 2-gram 有： he, el, ll, lo

      m、E对共享q-gram数的影响：Given a sequence of length m, the minimum number of shared q-grams between two similar sequences is therefore [m-q+1, q*E] .

      特性：对于e编辑敏感，将破坏多个q-gram（q-gram filtering is generally robust in handling only a small number of edits, as the presence of edits in any q-gram is significantly underestimated (e.g., counted as a single edit). ）

      【GRIM-Filter[^7]】

    • sparse DP

      仅在种子之间的间隙区域（非精确匹配区域）执行DP计算（种子本身是完全匹配的）

  • ACCELERATING SEQUENCE ALIGNMENT

    长read对DP算法计算复杂度提升大，此时用到 Heuristic-Based Alignment Accelerators

    • Accurate Alignment Accelerators

      SIMD CPUs【Parasail[^8] and KSW2 (used in mini-map2[^2])，加速diagonal bands计算】、多核CPUGPU【GASAL2，面向GPU碱基不采用二进制编码，较Parasail快20x，但索引结构很大】、FPGA【脉冲阵列a linear systolic array，加速DP算法，FPGASW[^13]、Darwin[^14]】、ASICs【GenAx[^9]】、PIM架构【computation inside or near the memory chip，RAPID[^10]】

    • Heuristic-Based Alignment Accelerators

      通过限制算法灵活性提升速度。如不执行回溯操作backtracking、限制评分函数的灵活性，即使用固定的评分算法（Levenshtein distance and Myers’s bit-vector algorithm【ASAP[^11]】、Edlib[^12]），但都可能导致次优解。

      限制DP算法的输入数：1）sparse DP；2）“using a greedy approach to maintain a high alignment score”（按alignment score减少DP计算输入和提前结束【X-drop[^15]、Z-drop】）

一、整体介绍

  • 全流程

    ![Overview of a typical genome analysis pipeline](基因分析课题/Overview of a typical genome analysis pipeline.png)

    1. Genomic sequencing data is first obtained through sequencing a new sample, downloading from publicly-available databases, or computer simulation. Sequencing starts with (A) extracting the DNA, (B) fragmenting it, (C) preparing its library, and (D) using sequencing machines for providing raw sequencing data.

    2. Raw sequencing data needs to be converted into read sequences of A, C, G, T in the DNA alphabet using basecalling techniques. Basecalling techniques are sequencing technology dependent.

    3. To ensure high quality sequencing reads, a quality control step is performed to filter out low quality subsequences of a read or an entire read sequence.

    4. Read mapping step is performed to locate each read sequence in a reference genome. Read mapping is four steps: (A) indexing the reference genome, (B) extracting seeds from each read and locating common seeds with the index, (C) pre-alignment filtering dissimilar sequence pairs, and (D) performing sequence alignment for every sequence pair that passes the filtering. （DP）

    5. Detecting and inferring genomic variations usually consists of three steps: (A) processing the read mapping data for increasing its quality, (B) classifying variations between mapped reads and a reference genome, and (C) identifying genomic variations.

**chaining**—— 稀疏DP过滤技术通过将重叠的种子（overlapping seeds）连接起来构建更长的链（longer chains），并以计算链的总长度（total length）作为序列对的过滤指标，该方法用于加速预对齐过滤。

【used in **minimap2** [130] and **rHAT** [176]】

【**mm2-fast** [162]——accelerates minimap2’s chaining step by up to 3.1⨉ with **SIMD instructions**.】

【**mm2-ax** [178]——accelerates mm2-fast’s chaining step by up to 12.6⨉ using a **GPU**.】

【**Modular Aligner** [179]—— 提出了一种基于两种种子过滤技术的chaining替代方案，在准确性和运行时间方面均实现了优于minimap2的性能表现】

  • Accelerating Sequence Alignment

  • 硬件加速最优仿射间隙得分（optimal affine gap scoring DP-based algorithms）

    算法：

    • 经典仿射间隙得分基于【Smith-Waterman-Gotoh algorithm [180]】

    • 【 the fastest known edit distance algorithm [181] has a nearly quadratic running time,O(m2/log2 m) 】

    • 【带状算法Ukkonen’s banded algorithm[182]. ——only a subset of the entries of the DP table is computed, called diagonal vectors（对角向量）, as first proposed—— This reduces the runtime complexity to O(m*E). ——对短read有效】

    硬件实现：

    • (1) using SIMD-capable CPUs,

      【Parasail [183] and KSW2 (used in minimap2 [130])——加速带状算法（Ukkonen’s banded algorithm）】

      【mm2-fast [162]——accelerates KSW2 by up to 2.2⨉ by matching its SIMD capability to recent CPU architectures 】

      【The wavefront algorithm (WFA) [184]——reformulates the classic Smith-Waterman-Gotoh recursion—— the runtime is reduced to O(m*s)】

      【LEAP [187] formulates what can be considered a more general version of WFA, which is applicable to any convex penalty scores（凸罚分函数）.】

    • (2) using multicore CPUs and GPUs,

      【GASAL2 [188]——transfers the bases to the GPU, without encoding them into binary format（不进行二进制编码）, hides the data transfer time by overlapping GPU and CPU execution（操作时间覆盖）——20⨉ faster than Parasail (executed with 56 CPU threads 】

      【BWA-MEM2 [158]——accelerates the banded sequence alignment of its predecessor (BWA-MEM [189]) by up to 11.6⨉, by leveraging multicore and SIMD parallelism.】

      【A GPU implementation [190] of the WFA algorithm improves the original CPU implementation by 1.5-7.7⨉ using long reads. 】

    • (3) using FPGAs,

      【FPGASW [191]——hardware execution units to form a linear systolic array [192].——a similar execution time as its GPU implementation, but 4⨉ more power efficient. 】

      【SeedEx [193]—— similar to FPGASW——speculatively computing fewer than 2E+1 diagonal bands（推测性地计算条带）, and then applying optimality checking heuristics（最优性验证启发方法） to guarantee correct results.】

      【 An FPGA accelerator [194] can accelerate the WFA algorithm by up to 8.8⨉ and improve its energy efficiency by 9.7⨉ for only short reads. 】

    • (4) using ASICs（Specialized hardware accelerators）

      【GenAx [195]——SillaX（a sequence alignment accelerator）的组成部分—— GenAx is 31.7⨉ faster than BWA-MEM [189] for short reads. 】

    • (5) using processing-in-memory architectures.

      【RAPID [196]——calculating two DP matrices (similar to Smith-Waterman-Gotoh [180]): one for calculating substitutions and exact matches and another for calculating insertions and deletions.（计算两个DP矩阵：一个用于计算替换和精确匹配，另一个用于计算插入和删除）—— RAPID is on average 11.8⨉ faster and 212.7⨉ more power efficient than 384-GPU cluster implementation of sequence alignment, known as CUDAlign [197]. —— A recent PIM architecture of WFA algorithm implemented in real hardware provides up to 4.87⨉ higher throughput than the 56-thread CPU implementation using short reads [198]. 】

  • Alignment Accelerators with Limited Functionality

developing heuristics that sacrifice the optimality of the alignment score solution in order to reduce alignment time.——减少DP矩阵计算条目

- limiting the **scoring function** (e.g. allowing only linear gap or unit scores)

  calculating only the alignment score **without performing the backtracking step** [199].

  【**Myers’ bit-vector algorithm [200]**—— the scoring function is limited to edit distance [165]】

  【**BitPAl [201]**—— limiting the scoring function to linear gap scores and achieves speedups through bit-parallel execution（ 将评分函数限制为线性间隙得分，并行执行加速。 ）】

  【 **ASAP [202]** ——accelerates edit distance calculation by up to 63.3⨉ using FPGAs compared to its CPU implementation. 】

  【**Edlib [131]**——fixed scoring function——the state-of-the-art implementation of Myers’ bit-vector algorithm——outperform **Parasail** (uses a **flexible** scoring function) by 12–1000⨉. 】

  缺点：may lead to the selection of a **suboptimal sequence alignment**（可能选到次优序列）

- trading off accuracy（降低精确度）

  【**Darwin** [36]—— by dividing the DP matrix into overlapping **submatrices** and greedily processing each submatrix using **systolic arrays**.—— 与**Edlib[131]**相比，快了三个数量级 】

  【**GenASM** [35]——use **bit-vector-based ASM** and a similar strategy as **Darwin**（与Darwin类似结构——implemented inside 3D-stacked memory】

- 其他

  -  using sparse DP 

  -  using a greedy approach to maintain a high alignment score. 

    【**X-drop** [208]—— (1) avoids calculating entries (and their neighbors) whose alignment scores are **more than below the highest score** seen so far (where is a user-specified parameter), and (2) **stops early** when a high alignment score is not possible. ——【 **A recent GPU implementation [209]** of the X-drop—— 3.1–120.4⨉ faster than KSW2 】】

    【Z-drop——makes **KSW2** at least 2.6⨉ faster than **Parasail** 】

    【**adaptive banding [210]**（ its improved algorithm **Block aligner [211]** ）——根据最近计算的反对角线中的最高分，计算对角线带会向上或向下移动。】
